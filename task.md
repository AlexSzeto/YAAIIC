# Ambient Brew Editor Changes, Round 1

## Goal
Update the Ambient Brew Editor to fit original design intent.

## Tasks

- [] Create `server/scripts/migrate/2-backfill-media-types.mjs` — a standalone Node.js script that reads `server/database/media-data.json`, iterates every entry that is missing a `type` field, assigns `type: 'audio'` if the entry has an `audioUrl`, assigns `type: 'video'` if its `imageUrl` ends with `.webp`, and otherwise assigns `type: 'image'`, then writes the file back to disk. Add `BREW_DATA_PATH` to `server/core/paths.mjs` to keep path resolution consistent.
- [] In `public/js/app-ui/brew-editor/sound-source-form.mjs`, change the clip data format from a plain string to an object `{ url: string, label: string }`. Update `createItem` to return `{ url: '', label: '' }`, update `handleGallerySelect` to populate both `url` (from `entry.audioUrl`) and `label` (from `entry.name`), and render a read-only or disabled text `Input` showing the label instead of the raw URL. Update `ambient-coffee.js` `load()` to extract the `url` property when a clip is a `{ url, label }` object (the existing string and range-object cases must continue to work).
- [] In `public/js/app-ui/brew-editor/sound-source-form.mjs`, add a compact inline audio preview control to each clip row: an audio player custom-ui component pointing to the clip's `url`. Only one clip should play at a time (stopping the previous one when a new one is started).
- [] Add sound-source-level preview to `public/js/app-ui/brew-editor/sound-source-form.mjs`. Accept an `onPreview(tempBrew)` prop (called with a fully-formed brew object) and a `isPlaying` prop. Add a Preview/Stop `Button` that calls `onPreview` with a temporary brew containing only this source and one auto-generated channel: `{ distance: 'medium', muffled: false, reverb: false, tracks: [{ label: 'Preview', type: 'event', clones: 1, sources: [source.label], delay: { min: 600, max: 600 }, delayAfterPrev: false }] }`. In `brew-editor.mjs`, wire the `onPreview` prop on each `SoundSourceForm` to the existing `startPreview` / `stopPlayback` mechanism by accepting the temporary brew instead of the full brew.
- [] Add channel-level preview to `public/js/app-ui/brew-editor/channel-form.mjs`. Accept an `onPreview(tempBrew)` prop and `isPlaying` prop. Add a Preview/Stop `Button` that calls `onPreview` with a temporary brew containing all sources from the current brew but only this single channel. In `brew-editor.mjs`, pass all brew sources down through `ChannelForm` and wire the `onPreview` prop to the same play/stop mechanism.
- [] Minimize vertical space in the brew editor forms by replacing standalone `widthScale="full"` inputs with pairs of inputs inside `HorizontalLayout` wrappers using the default (`"normal"`) width scale across `brew-editor.mjs`, `channel-form.mjs`, `sound-source-form.mjs`, and `track-form.mjs`. Group logically related fields side-by-side: e.g., Label + Distance in `ChannelForm`, Muffled + Reverb toggles in `ChannelForm`, Label + Type in `TrackForm`, Clones alongside its adjacent control.
- [] In `server/features/upload/service.mjs`, set `type: 'audio'` on the `generationData` object for audio uploads before calling `addMediaDataEntry`. Set `type: 'image'` for image uploads in the same way, so the Gallery `fileTypeFilter` can correctly filter entries by media type.
- [] In `server/features/brew/service.mjs` and `router.mjs`, replace the per-file storage pattern (one `.json` per brew in `server/database/brews/`) with a single flat `server/database/brew-data.json` file that stores all brews as an array, mirroring the structure of `media-data.json`. Update all CRUD operations (`listBrews`, `loadBrew`, `saveBrew`, `deleteBrew`) to read/write this single file. Migrate the existing brew files in `server/database/brews/` into the new format on first load, then remove the `brews/` subdirectory.
- [] In `public/js/app-ui/hamburger-menu.mjs`, add a "Brew Editor" navigation item pointing to `brew-editor.html`, so the brew editor is reachable from the global navigation. In `public/js/app-ui/brew-editor/brew-editor.mjs`, import and render `HamburgerMenu` inside the `AppHeader` alongside the `H1`, consistent with how other pages use it.
- [] In `public/js/app-ui/brew-editor/brew-editor.mjs`, remove the standalone `New`, `Import`, and `Export` header buttons. Move their functionality into the `ListSelectModal` (Open) flow: Import and Export should be accessible as actions within that modal or as secondary actions on the Open button, keeping the header uncluttered.
- [] In `public/js/app-ui/brew-editor/brew-editor.mjs`, remove the "Media URL (base path for clips)" `Input` field from the Brew Settings panel and remove the `mediaUrl` property from `createDefaultBrew`. Clips already resolve via the gallery and do not need a base URL override.
- [] In `public/js/app-ui/brew-editor/sound-source-form.mjs`, remove the `Input` for "Audio path or URL" from the clip `renderClipItem` renderer and remove the `handleClipTextChange` handler. Clips should only be set via the Browse/Gallery button, making the raw text field redundant and error-prone.
